## 三、数据来源深度复盘

### 3.1 数据演变史

```
阶段0: 初始 (硬编码) ── highway_routes.py
        └── 人手编写的静态坐标点, 各线点密度极低 (11~154点)
        
阶段1: OSM Overpass 提取 ── fetch_hw_osm.py
        └── 全要素提取(motorway+trunk) by 1°×1° 区块
        └── 过滤湖南行政区内
        └── 按 ref 归类 (G4,G55等)
        └── 问题: Overpass API 不稳定/超时, 大量断线
        
阶段2: OSM osmium 官方案 ── shell pipeline
        └── `osmium tags-filter`
        └── `osmium export` GeoJSON
        └── `shapely linemerge` 解决断线连通
        └── 坐标变换 (OSM lon,lat→lat,lon)
        └── 下采样(自动去冗余)
        └── 产出大部分精品数据
        
阶段3: 融合修复 ── merge_data.py
        └── JSO <40点 且 PY 更优 → 从PY合并
        └── 统一插密化 (max 3km→退出)
        └── 产出 highway_routes.json (当前主数据源)
```

## 四、多渠道数据收集方案研究

### 4.1 OSM 深度提取方案 (当前已是最优自由数据源)

**推荐增强策略: Route Mastery**
代替当前 `way` 级别的 bbox 查询，使用 `relation` 级别的线路级查询精确获取。
```
[relation] 提取: relation["ref"="G4"]["network"="CN:GH"](area.a3600062491);
```

**OSM 当前高质量提取 (已成功的16条) 对比 低质量3条的差异分析:**

| 情形 | 原因 | 解决方案 |
|------|------|----------|
| 国高G4/G56/G65 | OSM 原始关系不连续(省界穿走/N重tag歧义/ref别名) | 关系master+多段合并 |
| 省高S80 | 线路段存在但 OSM 中无统一ref标注 | 通过名"衡邵高速"匹配+手动确认 |

**OSM 提取改进方案:**
```python
# Overpass QL 高级查询 - 按关系提取完整几何
query = '''
[out:json];
(
  relation["route"="road"]["ref"~"G4"]["name"~"京港澳"](area.a3600062491);
  relation["route"="road"]["ref"~"G56"]["name"~"杭瑞"](area.a3600062491);
  relation["route"="road"]["ref"~"G65"]["name"~"包茂"](area.a3600062491);
);
out body;
>;
out geom;
```

**本地 osmium 流水线升级:**
```bash
# osmium 官方推荐 pipeline
osmium getid -r malaysia_continent.pbf r123456  -t \
  | osmium tags-filter - highway=motorway -t \
  | osmium export -f geojson -o route.json
```

### 4.2 商业地图API方案 (建议作为精校/实时层)

#### 4.2.1 百度地图API
- **服务**: 矩形路径规划 / 静态路线规划 / 批量获取
- **精度**: 车道级轨迹, 可返回完整坐标序列
- **限制**: 有免费额度, 企业级需付费(约0.5-2元/千次)
- **优势**: 国内道路最权威, 更新最快, 含实时施工/改道
- **API示例**: `http://api.map.baidu.com/directionlite`

#### 4.2.2 高德地图API
- **服务**: 路径规划+批量地理编码
- **精度**: 类似百度
- **优势**: 高速/城市道路精细度有时优于百度在部分区域
- **API**: `https://restapi.amap.com/v5/direction/driving`

#### 4.2.3 天地图 (国家地理信息公共服务平台)
- **权威性**: 国家级, 公路交通数据由交通部/自然资源部联合维护
- **数据**: https://www.tianditu.gov.cn/ → 道路矢量层
- **优势**: 绝对权威, 但访问需要申请 KEY, 政策审批可能耗时

### 4.3 自动补全/验证方案

**Web-Scraper 备选 (不推荐作为主源):**
- 从技术法律上讲，从地图网页提取并不理想（ToU 禁止），但学术/纯本地项目较常见
- 时效性和可用性极差（对抗干扰）

**GPS-OpenData 聚合层:**
- GPX 运动聚合平台 (Strava heatmap, 匿名化不上台面)
- AWS OpenStreetMap 公共数据集

## 七、技术结论

1. **整体评价**: 大数据(16/24条=67%)达到A级精品级, OSM路线数据真实可靠。
2. **核心短板**: G4/G56/G65三条通省大动脉为低质量的插值数据, 是项目的"致命伤"和首要优化方向。
3. **修复确定性**: 高 - 路线关系在OSM中极大概率存在, 只需从way级提升到route级关系提取即可。
4. **无需焦虑**: 绝大多数(21/24)数据已经高质量, 不会因为3条而否定整体。

**下一步推荐**: 先对 G4(京港澳)做一段OSM关系级提取试验, 看是否能获得600+点级的精准路线。成功后平推G56/G65。
